# Getting Started

Shortest path from a fresh Debian VPS to a running Cool Tunnel Server: Docker Compose, Caddy, sing-box, VLESS + Reality, Bun/Hono/Better Auth admin, SQLite admin auth storage, and the `ct` operator CLI.

## Requirements

| Need | Notes |
| --- | --- |
| Debian 12 or newer VPS | Debian 12 is the primary target |
| Root SSH or sudo | Needed for Docker and firewall setup |
| A domain pointed at the VPS | Set an `A` record to the VPS public IPv4 |
| Ports `80/tcp` and `443/tcp` open | Caddy uses 80 for ACME and 443 for panel/proxy traffic |
| Docker Engine + Compose v2 | The installer checks this before loading release images |

## Install

```sh
apt update && apt -y upgrade
apt install -y ca-certificates curl gnupg ufw dnsutils chrony fail2ban unattended-upgrades git
ufw allow OpenSSH
ufw allow 80/tcp
ufw allow 443/tcp
ufw --force enable

LATEST="$(curl -fsSLI -o /dev/null -w '%{url_effective}' https://github.com/coo1white/cool-tunnel-server/releases/latest | sed 's#.*/##')"
BRANCH="${LATEST}" /bin/bash -c "$(curl -fsSL "https://raw.githubusercontent.com/coo1white/cool-tunnel-server/${LATEST}/scripts/bootstrap.sh")"
cd /opt/cool-tunnel-server
$EDITOR .env
ct install
ct doctor
```

Set these keys in `.env` before install:

| Key | Value |
| --- | --- |
| `DOMAIN` | Proxy/base domain |
| `PANEL_DOMAIN` | Admin panel hostname, usually `panel.<DOMAIN>` |
| `ACME_EMAIL` | Email for certificate renewal notices |
| `DB_ROOT_PASSWORD` | Long random secret, if MariaDB is retained |
| `DB_PASSWORD` | Different long random secret, if MariaDB is retained |
| `REDIS_PASSWORD` | Different long random secret, if Redis is retained |
| `BETTER_AUTH_SECRET` | Generated by install/migration when missing |

The installer downloads verified Docker image slices, runs SQLite admin/auth migrations, renders Caddy and sing-box config, and starts the stack. Normal tagged releases do not build Rust, Bun, Go, or Docker images locally.

## First Owner

There is no default admin password. Create the first owner with a one-time token:

```sh
cd /opt/cool-tunnel-server
ct admin bootstrap
```

`ct admin bootstrap` writes the setup page and one-time token to a root-only file and prints the exact `sudo cat ...` command to read it over SSH. Open the setup page, paste the token, choose the owner email/name/password in the browser, delete the file, then sign in at:

```text
https://<PANEL_DOMAIN>/admin
```

The token expires, can be used once, and bootstrap disables automatically after an owner exists.

## Update Later

```sh
cd /opt/cool-tunnel-server
ct backup
ct update
ct doctor
```

If the local checkout is stale or messy, inspect it before discarding edits. On a normal production VPS where source edits are not intended:

```sh
git status -sb
git fetch origin
git checkout main
git reset --hard origin/main
./scripts/fetch_operator_binary.sh || true
ct update
ct doctor
```

## Verify

```sh
ct doctor
docker compose ps
docker compose logs --tail=120 panel
docker compose logs --tail=120 caddy
docker compose logs --tail=120 singbox
```

If `ct install` or `ct update` says the prebuilt Docker image bundle is missing, the release is incomplete for that CPU architecture. Fetching manually should either load it or print the exact missing asset/checksum:

```sh
./scripts/fetch_image_bundle.sh
ct update
```

## What Runs

| Service | Role |
| --- | --- |
| `caddy` | Public front door, ACME, and SNI routing |
| `singbox` | User proxy service supervised by `singbox-core` |
| `panel` | Bun/Hono/Better Auth admin UI and Rust-boundary commands |
| `db` | MariaDB store for retained core runtime data |
| `redis` | Runtime cache/compatibility service where still required |

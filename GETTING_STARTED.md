# Getting Started

Shortest path from a fresh Debian VPS to a running Cool Tunnel Server:
an open-source self-hosted proxy server using Docker Compose, Caddy,
sing-box, VLESS + Reality, a Next.js admin UI, a Bun/Hono API, Better
Auth, SQLite, and the `ct` operator CLI. For the full Debian VPS
installation reference, use
[`docs/installation-debian.md`](./docs/installation-debian.md).

## Requirements

| Need | Notes |
| --- | --- |
| Debian 12 or newer VPS | Debian 12 is the primary target |
| Root SSH or sudo | Needed for Docker and firewall setup |
| A domain pointed at the VPS | Set an `A` record to the VPS public IPv4 |
| Ports `80/tcp` and `443/tcp` open | Caddy uses 80 for ACME and 443 for panel/proxy traffic |
| Docker Engine + Compose v2 | The installer checks this before loading release images |

Before installing, confirm DNS from your laptop:

```sh
dig +short A proxy.example.com
```

The result should be the VPS public IP.

## Debian VPS install flow

Use the project this way:

```text
copy bootstrap command
edit .env
ct install
ct doctor
copy subscription URL into client
```

After that:

```text
ct backup
ct update
ct doctor
```

Avoid hand-editing generated Docker, Caddy, or sing-box files on the
VPS. The admin API and operator regenerate them so server, client, and
runtime plugins stay compatible.

## Install

SSH into the VPS, then:

```sh
apt update && apt -y upgrade
apt install -y ca-certificates curl gnupg ufw dnsutils chrony fail2ban unattended-upgrades git
```

Open the firewall:

```sh
ufw allow OpenSSH
ufw allow 80/tcp
ufw allow 443/tcp
ufw --force enable
```

Either install Docker from Docker's official Debian repository yourself,
or let the bootstrap command below install it for you. To verify:

```sh
docker version
docker compose version
```

The detailed Docker steps live in
[`docs/installation-debian.md`](./docs/installation-debian.md#5-install-docker-engine--compose-v2).

Bootstrap the project with the same one-command shape
[Homebrew](https://brew.sh/) uses:

```sh
LATEST="$(curl -fsSLI -o /dev/null -w '%{url_effective}' https://github.com/coo1white/cool-tunnel-server/releases/latest | sed 's#.*/##')"
BRANCH="${LATEST}" /bin/bash -c "$(curl -fsSL "https://raw.githubusercontent.com/coo1white/cool-tunnel-server/${LATEST}/scripts/bootstrap.sh")"
cd /opt/cool-tunnel-server
$EDITOR .env
```

Set these keys in `.env`:

| Key | Value |
| --- | --- |
| `DOMAIN` | Proxy/base domain |
| `PANEL_DOMAIN` | Admin panel hostname, usually `panel.<DOMAIN>` |
| `ACME_EMAIL` | Email for certificate renewal notices |
| `BETTER_AUTH_SECRET` | Long random secret, for example `openssl rand -base64 48` |
| `REALITY_PRIVATE_KEY` | 43-character Reality private key |
| `REALITY_PUBLIC_KEY` | Matching 43-character Reality public key |

Install:

```sh
ct install
```

The installer downloads the verified Docker image bundle for your CPU
architecture, loads it with Docker, migrates the DB, renders Caddy
and sing-box config, starts the stack, and creates the first admin
setup path. On normal tagged releases, a small VPS does not build Rust,
Bun, Go, Next.js, or Docker images locally.

## Update Later

Normal update:

```sh
cd /opt/cool-tunnel-server
ct backup
ct update
ct doctor
```

Recovery update when the local checkout is stale or messy:

```sh
cd /opt/cool-tunnel-server
git fetch origin
git checkout main
git reset --hard origin/main
./scripts/fetch_operator_binary.sh || true
ct update
ct doctor
```

`git reset --hard origin/main` discards tracked local source edits. Use
it only on a normal production VPS where the repo should be clean.

## Panel Login and Account Setup

Open the admin UI:

```text
https://<PANEL_DOMAIN>/login
```

Create the first owner from the VPS. The bootstrap token is one-time
only and expires:

```sh
cd /opt/cool-tunnel-server
ct admin bootstrap
```

Open the printed setup URL, create the owner account, then log in at
`/login`. Public signup is disabled by default. If you need to
recover an admin from the VPS:

```sh
cd /opt/cool-tunnel-server
printf '%s\n' '<new long password>' | ct admin create-owner --email you@example.com --username you --password-stdin
printf '%s\n' '<temporary long password>' | ct admin users reset-password --id <user-id> --password-stdin
```

Create a client account:

```text
Users -> New proxy account -> Save
```

After saving, open the account row's **Subscription URL** action and
copy the **Import URL** into the Cool Tunnel client. The URL has this
shape:

```text
https://<PANEL_DOMAIN>/api/v1/subscription/<token>
```

The subscription URL contains the per-account import token. Treat it
like a password; do not paste it into public tickets or logs. If you
regenerate the UUID, copy the fresh Subscription URL after rotation.

The imported client profile uses the VLESS+Reality configuration
generated by the admin API and `singbox-core`.

## Verify

Run the local health gate:

```sh
ct doctor
```

If doctor reports WARN or FAIL, follow the remediation text first.
Useful quick checks:

```sh
docker compose ps
docker compose logs --tail=120 caddy
docker compose logs --tail=120 singbox
docker compose logs --tail=120 admin-api
docker compose logs --tail=120 admin-web
```

If `ct install` or `ct update` says the prebuilt Docker image bundle is
missing, the release is incomplete for that CPU architecture. Fetching
the release images manually should either load them or print the exact
missing asset/checksum:

```sh
./scripts/fetch_image_bundle.sh
ct update
```

For operations after install, use
[`docs/operator-runbook.md`](./docs/operator-runbook.md) and
[`docs/operations.md`](./docs/operations.md).

## What Runs

| Service | Role |
| --- | --- |
| `caddy` | Public front door, ACME, and SNI routing |
| `singbox` | User proxy service supervised by `singbox-core` |
| `admin-api` | Hono/Bun API, Better Auth, SQLite store, subscription endpoint, and render commands |
| `admin-web` | Next.js admin dashboard |

See [`STRUCTURE.md`](./STRUCTURE.md) for the code map.

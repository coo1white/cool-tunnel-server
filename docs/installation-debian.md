# Debian Installation Reference

This is the longer reference for a fresh Debian VPS install. The short path is in `GETTING_STARTED.md`.

## Install Flow

```sh
apt update && apt -y upgrade
apt install -y ca-certificates curl git gnupg ufw dnsutils chrony fail2ban unattended-upgrades
ufw allow OpenSSH
ufw allow 80/tcp
ufw allow 443/tcp
ufw --force enable

LATEST="$(curl -fsSLI -o /dev/null -w '%{url_effective}' https://github.com/coo1white/cool-tunnel-server/releases/latest | sed 's#.*/##')"
BRANCH="${LATEST}" /bin/bash -c "$(curl -fsSL "https://raw.githubusercontent.com/coo1white/cool-tunnel-server/${LATEST}/scripts/bootstrap.sh")"
cd /opt/cool-tunnel-server
$EDITOR .env
ct install
ct doctor
ct admin bootstrap
```

`ct admin bootstrap` writes the setup page and one-time token to a root-only file and prints the exact `sudo cat ...` command to read it over SSH. Open the setup page, paste the token, create the first owner, then delete the file. There is no default admin password.

## Required Config

Set at least:

```dotenv
DOMAIN=proxy.example.com
PANEL_DOMAIN=panel.proxy.example.com
ACME_EMAIL=ops@example.com
BETTER_AUTH_SECRET=<generated by install if missing>
CT_PUBLIC_SIGNUP=false
CT_ADMIN_DB_PATH=/data/admin/admin.sqlite
```

MariaDB and Redis secrets are still required when retained runtime services use them.

## Runtime

| Service | Role |
| --- | --- |
| `caddy` | ACME and SNI routing |
| `panel` | Bun/Hono/Better Auth admin server |
| `singbox` | VLESS + Reality proxy |
| `db` | MariaDB for retained core runtime data |
| `redis` | Retained runtime compatibility/cache service |

## Troubleshooting

```sh
ct doctor
docker compose ps
docker compose logs --tail=120 panel caddy singbox
ct render caddyfile
ct render singbox
```

If install/update reports a missing image bundle, publish the release bundle for that CPU architecture or rerun:

```sh
./scripts/fetch_image_bundle.sh
ct update
```

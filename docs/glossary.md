# Glossary

Core terms used in the current Cool Tunnel Server stack.

## ACME

The certificate automation protocol used by Let's Encrypt. Caddy uses
ACME to issue the panel certificate.

## BETTER_AUTH_SECRET

The admin API secret used by Better Auth and by Cool Tunnel's signed
bootstrap/subscription tokens. Rotating it logs admins out and requires
fresh subscription URLs for clients.

## Caddy

The public front door. The built image includes `mholt/caddy-l4`, so
Caddy listens on `:443`, routes panel SNI to the admin web app, and
forwards non-panel SNI traffic to sing-box. It also handles ACME for
the panel domain.

## Component Pins

Version records for replaceable runtime pieces. See
`docs/components.md` for the current source-of-truth files and health
gates.

## ct-protocol

The shared Rust crate (`core/ct-protocol`) holding the wire, profile,
and subscription-manifest types. The server is its canonical source;
every client fetches a matching version from the published release tag.

## docker compose

The runtime orchestrator for the four long-running services: `caddy`,
`singbox`, `admin-api`, and `admin-web`.

## .env

The deployment config and secret file at the repo root. It contains
domains, Better Auth secret, Reality keys, SQLite path, and related
operator settings.

## Account Admin

The Bun/Hono admin API and Next.js admin web app for login, first-owner
setup, user management, proxy accounts, role checks, settings, status,
and audit history.

## Better Auth

The authentication library used by `apps/api` for email/password login,
sessions, rate limiting, and secure cookies. Public signup is disabled
by default.

## GFW

The Great Firewall of China. See `docs/restrictive-networks.md` for the
operator runbook.

## PASS / WARN / FAIL

The status labels used by `ct doctor`. PASS is healthy, WARN needs
attention soon, and FAIL should be fixed before trusting the release.

## admin-api

The Bun/Hono API container. It owns auth, RBAC, SQLite migrations,
subscription output, status, audit history, and render action
boundaries.

## admin-web

The Next.js dashboard container. It serves the operator UI and talks to
`admin-api` over the internal Compose network.

## PANEL_DOMAIN

The hostname for the admin UI, usually `panel.<DOMAIN>`.

## Reality

The sing-box TLS camouflage mode used by the VLESS proxy path.

## sing-box

The proxy engine. It runs in the `singbox` container under
`singbox-core supervise`, reads `/data/config/singbox.json`, and is
restarted when the rendered config changes.

## SNI

Server Name Indication, the hostname inside the TLS ClientHello. Caddy
uses SNI to route panel traffic separately from proxy traffic.

## Subscription Manifest

The signed JSON document the admin API returns at a per-user subscription
URL. Clients use it to configure server host, port, UUID, and sing-box
pin data.

## SQLite

The default control-plane database. It is stored in the
`./data/admin/admin.sqlite` on the host and bind-mounted into admin-api
as `/data/admin/admin.sqlite`.

## VLESS

The sing-box protocol used for end-user proxy accounts. Each account
uses a UUID generated by the admin API.
